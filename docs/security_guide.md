# ResearchReel — Security Architecture & Validation Guide

This document defines the security architecture, data-flow boundaries, implementation details, and verification checks configured to secure the **ResearchReel** platform.

---

## 1. Security Architecture Overview

ResearchReel follows a defense-in-depth model, securing the platform at the network, application, database, and container layers.

```text
       [ Public Client ]
               │  HTTPS (TLS 1.3)
               ▼
      [ Cloudflare CDN ]  (DDoS Protection, WAF)
               │
               ▼
    [ Nginx Ingress Controller ]  (SSL/TLS Termination)
               │  HTTP
               ▼
      [ API Gateway Pod ]  (Rate Limiting, JWT Validation, Helmet)
        │            │
        │ gRPC/HTTP  │ gRPC/HTTP
        ▼            ▼
   [ Auth Svc ]  [ Reel Svc ]  (NetworkPolicies restrict service boundaries)
        │            │
        ▼            ▼
  [ PostgreSQL ]  [ AWS S3 ]
```

## 2. Core Application Security Controls

These controls are implemented and verified in the codebase (principally in the `backend/src/` gateway and service components).

### 2.1 HTTPS (SSL/TLS)
* **Status**: Configured.
* **Mechanism**: Handled by Nginx Ingress Controller (`k8s/ingress.yml`) using `cert-manager` for automatic Let's Encrypt certificates.
* **Policies**: Restricts communication to TLS 1.2 and TLS 1.3, disabling obsolete ciphers (e.g., RC4, 3DES) and older TLS versions (1.0, 1.1).

### 2.2 JWT Stateless Authentication
* **Status**: Implemented and verified.
* **Mechanism**: Tokens are generated by the Auth Service and verified in [authMiddleware.js](file:///g:/Desktop/RESEARCHAPP/backend/src/middleware/authMiddleware.js).
* **Storage**: Access and Refresh tokens are delivered to the client via `HttpOnly`, `Secure`, `SameSite=Strict` cookies to mitigate Cross-Site Scripting (XSS) extraction risks.
* **Verification Proof**:
  ```javascript
  const token = req.cookies.accessToken || req.headers.authorization?.split(' ')[1];
  if (!token) return res.status(401).json({ success: false, message: 'No authentication token' });
  const decoded = jwt.verify(token, process.env.JWT_SECRET);
  req.user = decoded;
  ```

### 2.3 Password Hashing (Argon2)
* **Status**: Implemented and verified.
* **Mechanism**: Uses `argon2` inside [authService.js](file:///g:/Desktop/RESEARCHAPP/backend/src/services/authService.js).
* **Configuration**: Configured with standard enterprise parameters:
  * Memory cost: `65536 KB` (64 MB)
  * Time cost: `3` iterations
  * Parallelism: `4` threads
* **Verification Proof**:
  ```javascript
  const hashedPassword = await argon2.hash(password);
  // Verification:
  const isMatch = await argon2.verify(hashedPassword, plainPassword);
  ```

### 2.4 SQL Injection Protection
* **Status**: Implemented and verified.
* **Mechanism**: Relational queries in PostgreSQL use parameterized queries. Native string interpolation (`${val}`) or concatenation (`+ val`) is forbidden for SQL generation.
* **Verification Proof**:
  ```javascript
  // Safe parameterized query
  const queryText = 'SELECT * FROM users WHERE email = $1';
  const result = await db.query(queryText, [email]);
  ```

### 2.5 Cross-Origin Resource Sharing (CORS)
* **Status**: Implemented and verified.
* **Mechanism**: Enforced via the `cors` package in Express ([app.js](file:///g:/Desktop/RESEARCHAPP/backend/src/app.js)).
* **Configuration**: Origin is strictly bound to `process.env.FRONTEND_URL` (no wildcard `*` allowed in production) and `credentials: true` is enabled.

### 2.6 Security Headers (Helmet)
* **Status**: Implemented and verified.
* **Mechanism**: Express gateway uses the `helmet` middleware.
* **Headers Injected**:
  * `Content-Security-Policy` (CSP): Mitigates XSS by restricting allowed script execution paths.
  * `X-Frame-Options`: Set to `DENY` to prevent clickjacking.
  * `Strict-Transport-Security` (HSTS): Enforces browser-side HTTPS redirects.
  * `X-Content-Type-Options`: Set to `nosniff` to prevent MIME-sniffing.

### 2.7 Rate Limiting
* **Status**: Implemented and verified.
* **Mechanism**: Implemented in [rateLimiter.js](file:///g:/Desktop/RESEARCHAPP/backend/src/middleware/rateLimiter.js) using Redis.
* **Rules**: 100 requests per 15 minutes per IP.
* **Resiliency**: If the Redis cluster crashes or becomes partitioned, the rate limiter logs the connection warning via Winston and **fails open** to prevent platform blackout for users:
  ```javascript
  try {
    const current = await redisClient.incr(key);
    // ... logic ...
  } catch (err) {
    logger.warn(`Rate limiter Redis error: ${err.message}. Failing open.`);
    next();
  }
  ```

### 2.8 Input Validation & Sanitization
* **Status**: Implemented and verified.
* **Mechanism**: Request body inputs are validated at the controller layer. Parameters are checked for email structure format, maximum text lengths (e.g. usernames < 30 chars), and integer bounds to prevent memory allocation or database overflow attacks.

### 2.9 XSS Prevention
* **Status**: Configured.
* **Mechanism**: Content-Security-Policy (CSP) headers restrict script injections. Client-side UI frameworks (Next.js/React) automatically escape dynamic text variables in JSX by default.

## 3. Secret Management & Injection

In production environments, hardcoded credentials or configuration secrets in git repositories are strictly prohibited.

| Secret Key | Destination | Production Store | Injection Mechanism |
| :--- | :--- | :--- | :--- |
| `jwt-secret` | API Gateway, Auth Svc | AWS Secrets Manager / Vault | Kubernetes Secret (`researchreel-secrets`) |
| `postgres-password` | PostgreSQL, Microservices | AWS Secrets Manager / RDS | Kubernetes Secret (`researchreel-secrets`) |
| `gemini-api-key` | API Gateway, AI Svc | AWS Secrets Manager | Kubernetes Secret (`researchreel-secrets`) |
| `qdrant-api-key` | AI/RAG Service | Cloud Qdrant Vault | Kubernetes Secret (`researchreel-secrets`) |
| `neo4j-password` | Citation Svc, Social Graph | Neo4j Aura DB / Vault | Kubernetes Secret (`researchreel-secrets`) |

### Example Kubernetes Secret Mapping:
```yaml
apiVersion: v1
kind: Secret
metadata:
  name: researchreel-secrets
  namespace: default
type: Opaque
stringData:
  jwt-secret: asdfghjklqwertyuiopzxcvbnmqwertyuiopasdfghjklzxcvbnm
```
*Mounted as environment variables inside pod definitions:*
```yaml
env:
  - name: JWT_SECRET
    valueFrom:
      secretKeyRef:
        name: researchreel-secrets
        key: jwt-secret
```

## 4. Kubernetes Network Isolation (Service Boundaries)

To prevent service-boundary violations (e.g., the User Profile Service direct querying the Auth database), Kubernetes `NetworkPolicies` isolate traffic inside the EKS cluster namespace.

### Default Deny Ingress Policy
By default, all pods reject incoming traffic unless explicitly permitted:
```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: default-deny-ingress
  namespace: researchreel
spec:
  podSelector: {}
  policyTypes:
    - Ingress
```

### Gateway Traffic Routing Rule
Permits only the Ingress controller to talk to the API Gateway, and the API Gateway to communicate with downstream microservices:
```yaml
apiVersion: networking.k8s.io/v1
kind: NetworkPolicy
metadata:
  name: allow-gateway-to-services
  namespace: researchreel
spec:
  podSelector:
    matchLabels:
      app: core-service # Targets backend microservice pods
  ingress:
    - from:
        - podSelector:
            matchLabels:
              app: api-gateway
      ports:
        - protocol: TCP
          port: 5000
```

## 5. OWASP Top 10 Mitigation Matrix

| Category | Vulnerability | ResearchReel Mitigations |
| :--- | :--- | :--- |
| **A01:2021** | Broken Access Control | HttpOnly, Secure, SameSite JWT validation; role-based access middleware checks path-level variables. |
| **A02:2021** | Cryptographic Failures | Argon2 password hashing; Transport TLS 1.3 encryption on all ingress paths; encryption at rest on EBS and RDS. |
| **A03:2021** | Injection | Strict parameterization of SQL queries (`pg`); validation schemas on document collections (MongoDB); input size checks. |
| **A04:2021** | Insecure Design | Monorepo modular separation, event-driven orchestration (Kafka) prevents microservice cascading failures. |
| **A05:2021** | Security Misconfiguration | Production Helm configurations strip default credentials; Helmet manages browser header security; CORS restrictions. |
| **A06:2021** | Vulnerable/Outdated Components | `npm audit` execution and automated security patches check during GitHub Actions execution (`deploy.yml`). |
| **A07:2021** | Identification/Authentication | Password lockouts; Redis rate-limit triggers preventing brute-force logins; JWT signature checks. |
| **A08:2021** | Software & Data Integrity | SHA hashes checked during container pull phases; GitHub Actions CI/CD runs builds in clean, isolated runners. |
| **A09:2021** | Security Logging & Monitoring | Winston JSON structured logging to stdout; Prometheus scraping alert rules configured for error spikes. |
| **A10:2021** | Server-Side Request Forgery | Microservices communicate via an isolated virtual network VPC; external APIs (Whisper/Gemini) are queried through secure proxies. |
